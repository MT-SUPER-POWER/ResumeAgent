# 《人才评级打分手册 V1》

> 转写说明：本文由原 DOCX 正文与图片表格整理为 Markdown。原文中大量内容以表格截图形式存在，以下已尽量识别并转写为 Markdown 表格。少数原图本身存在裁切，无法完整读取的地方已用说明标注，未凭空补全。

## 一、评分目的

本手册用于简历筛选阶段，评价候选人的**人才质量**，不直接判断其是否匹配某个具体岗位。

它回答的问题是：

> 这个人本身是不是高质量人才？是否值得优先关注、进入人才池或进入下一步评估？

人才评级结果用于：

| 用途           | 说明                                    |
| -------------- | --------------------------------------- |
| 简历初筛排序   | 在大量候选人中识别高质量人才            |
| 人才池分层     | S/A 人才进入重点人才池                  |
| 面试优先级     | 高等级人才优先安排面试                  |
| 岗位匹配联动   | 与岗位匹配度评分共同决定是否推进        |
| CEO/负责人关注 | S 级、部分 A 级候选人可进入高优先级复核 |

## 二、适用范围

| 项目     | 说明                                       |
| -------- | ------------------------------------------ |
| 适用阶段 | 简历筛选、人才池入库、候选人初评           |
| 适用对象 | 所有岗位、所有职级、所有候选人类型         |
| 不适用于 | 面试最终录用决策、薪资定级、绩效评估       |
| 评分依据 | 简历、作品集、公开资料、推荐信息等可见材料 |
| 评分原则 | 基于证据，不基于主观好感或过度推测         |

## 三、评分结构

人才评级总分为 **100 分**。

一级维度有不同权重；每个二级维度按 **0-10 分**评分，再按二级权重折算到一级维度，最后汇总为人才质量总分。

### 计算公式

```text
一级维度得分 = 二级维度加权平均分 ÷ 10 × 一级维度权重
人才质量总分 = 所有一级维度得分相加
```

### 一级维度总表

| 一级维度           | 一级权重 |
| ------------------ | -------: |
| 逻辑思维与认知能力 |      18% |
| 专业知识与专业技能 |      18% |
| 创新与 AI 原生能力 |      15% |
| 决策与问题解决能力 |      13% |
| 组织协作与低自我   |      12% |
| 抗压与韧性         |      10% |
| 职业规划与自驱力   |       8% |
| 履历质量与可信度   |       6% |
| 合计               |     100% |

## 四、通用 10 分评分标准

| 分数 | 等级       | 通用定义                                     |
| ---: | ---------- | -------------------------------------------- |
| 9-10 | 卓越       | 有强证据，表现明显优于同级候选人，具备稀缺性 |
|  7-8 | 良好       | 有明确证据，能独立完成高质量工作             |
|  5-6 | 一般       | 有一定证据，但复杂度、结果或影响力有限       |
|  3-4 | 较弱       | 只有弱信号，或描述模糊，不能充分证明能力     |
|  1-2 | 明显不足   | 有明显短板、负面信号，或与要求差距较大       |
|    0 | 严重不符合 | 完全不具备、严重不适配，或存在重大风险       |

## 五、证据等级规则

每个二级维度除了打 **0-10 分**，还要标记证据等级。

| 证据等级 | 定义                                                                       | 示例                              |
| -------- | -------------------------------------------------------------------------- | --------------------------------- |
| 强       | 有明确项目、数据、结果、角色、作品、代码、论文、专利、客户案例或第三方证明 | “主导 xxx 项目，使转化率提升 32%” |
| 中       | 经历描述较清晰，但结果数据、贡献边界或复杂度不完整                         | “负责推荐系统优化，提升用户体验”  |
| 弱       | 只有关键词、职责描述、泛化表达                                             | “熟悉 AI 工具”“具备沟通能力”      |
| 缺失     | 简历未体现该维度信息                                                       | 未提到相关内容                    |
| 负面     | 存在明显反向证据或风险                                                     | 频繁短期跳槽、成果夸大、表达混乱  |

### 缺失信息处理

| 情况     | 建议评分 |
| -------- | -------: |
| 强证据   |     8-10 |
| 中等证据 |      6-7 |
| 弱信号   |      4-5 |
| 未体现   |   默认 3 |
| 明显负面 |      0-2 |

注意：

> “未体现”不等于“中等水平”。
>
> 简历没有证据时，不能默认给 5 分。

## 八、二级维度评分手册

## 1. 逻辑思维与认知能力

### 1.1 结构化表达能力

| 项目      | 内容                                                                     |
| --------- | ------------------------------------------------------------------------ |
| 维度定义  | 候选人是否能用清晰结构表达经历、问题、行动和结果                         |
| 高分 8-10 | 简历结构清晰，项目描述有“背景-问题-行动-结果”，重点突出，逻辑顺畅        |
| 中分 5-7  | 简历基本清楚，但职责、行动、结果之间关系不够完整                         |
| 低分 1-4  | 简历堆砌经历，表达混乱，重点不清，只写职责不写结果                       |
| 缺失处理  | 简历本身即可作为判断依据，不单独缺失；表达极少时给 3-4                   |
| 正向证据  | 项目分层清楚、指标明确、表达简洁、有重点。原图下方被截断，剩余内容不可读 |
| 负向证据  | 原图下方被截断，内容不可读                                               |

### 1.2 问题拆解能力

| 项目      | 内容                                                           |
| --------- | -------------------------------------------------------------- |
| 维度定义  | 能否把复杂、模糊问题拆成目标、路径、模块、指标和优先级         |
| 高分 8-10 | 主导复杂项目拆解，如搭建系统、重构流程、制定策略，并有明确成果 |
| 中分 5-7  | 做过一定复杂度项目，能看到任务拆分，但方法论不够清晰           |
| 低分 1-4  | 主要是执行任务，缺少分析、判断、拆解信号                       |
| 缺失处理  | 默认 3                                                         |
| 正向证据  | “搭建指标体系”“重构流程”“制定策略”“拆解客户需求”“设计架构”等   |
| 负向证据  | 只写“参与、协助、负责日常工作”，没有具体行动和拆解过程         |

### 1.3 数据与因果思维

| 项目      | 内容                                                             |
| --------- | ---------------------------------------------------------------- |
| 维度定义  | 是否能用数据、实验、指标和因果关系判断问题                       |
| 高分 8-10 | 明确使用数据分析、实验、A/B 测试、模型评估、经营分析，并推动结果 |
| 中分 5-7  | 有数据意识和部分指标结果，但分析过程不完整                       |
| 低分 1-4  | 基本不体现数据判断，只描述主观行动                               |
| 缺失处理  | 默认 3                                                           |
| 正向证据  | 转化率、留存、准确率、成本、ROI、效率、实验组、评估指标          |
| 负向证据  | 只有“效果良好”“显著提升”等无数据表述                             |

### 1.4 系统性思维

| 项目      | 内容                                               |
| --------- | -------------------------------------------------- |
| 维度定义  | 是否理解业务、产品、技术、组织之间的相互关系       |
| 高分 8-10 | 能处理跨系统、跨模块、跨团队问题，方案兼顾多方约束 |
| 中分 5-7  | 能理解上下游协作，但主要在局部模块内发挥作用       |
| 低分 1-4  | 只关注单点任务，缺乏整体视角                       |
| 缺失处理  | 默认 3                                             |
| 正向证据  | 平台化、体系化、端到端、跨部门、架构、流程闭环     |
| 负向证据  | 只描述孤立任务，无上下游或整体结果                 |

### 1.5 跨领域理解力

| 项目      | 内容                                                |
| --------- | --------------------------------------------------- |
| 维度定义  | 是否能跨行业、跨职能、跨技术栈理解问题并迁移能力    |
| 高分 8-10 | 有跨领域成功迁移经历，并在新领域产生可验证成果      |
| 中分 5-7  | 有跨领域经历，但成果或深度有限                      |
| 低分 1-4  | 经历单一，或跨领域但没有体现学习和转化              |
| 缺失处理  | 默认 3                                              |
| 正向证据  | 技术+产品、产品+商业、研究+工程、行业迁移、复合背景 |
| 负向证据  | 频繁换方向但无积累、跨领域描述缺少成果              |

| 缺失处理  | 默认 3                                                       |
| 正向证据  | 专家角色、核心模块负责人、技术攻关、复杂客户/项目、深度研究  |
| 负向证据  | 经历广但浅，频繁切换方向无沉淀                               |

| 缺失处理  | 关键门槛缺失时保守给 3；明确不满足给 0-2                           |
| 正向证据  | 学历、工作许可、语言、地区、联系方式、基本信息完整                 |
| 负向证据  | 基础信息缺失、身份/地域/资质不满足、表达严重不专业                 |

## 九、评分输出格式

每位候选人建议输出以下字段：

| 字段         | 示例                                            |

| 人才质量总分 | 84.6                                            |

| 证据置信度   | 中                                              |
| 推荐动作     | 优先进入面试                                    |
| 面试验证点   | 验证关键项目真实贡献、AI 工具使用深度、抗压案例 |

## 十、评分纪律

| 规则                 | 说明                                             |
| -------------------- | ------------------------------------------------ |
| 不奖励空话           | “优秀、负责、抗压强、学习能力强”没有证据不加高分 |
| 不迷信大厂           | 大厂背景只是信号，不等于高人才评级               |
| 不惩罚非名校         | 非名校但有强作品、强成果、强成长，可以高分       |
| 不脑补能力           | 简历没有体现的能力，按缺失规则处理               |
| 不用单一缺点否定全部 | 除重大风险外，应按维度独立评分                   |
| 重视 AI 原生能力     | 2026 年后，能否用 AI 重构工作方式是重要区分点    |
| 重视低自我协作       | 高能力但高自我、低协作的人才评级不应过高         |

## 十一、最终评级建议

| 人才等级 | 建议动作                           |

| B        | 结合岗位匹配度推进                 |
| C        | 仅在岗位高度匹配时推进             |
| D        | 通常不推进，可入低优先级备选池     |
