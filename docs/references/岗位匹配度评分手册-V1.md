### 《岗位匹配度评分手册 V1》

### 一、评分目的

本手册用于简历筛选阶段，评价候选人与目标岗位之间的 **匹配程度**。

它回答的问题是：

| 这个人是否适合当前岗位？是否值得进入该岗位的面试流程？ |
| --- |


岗位匹配度不等同于人才质量。一个人可能人才评级很高，但不适合当前岗位；也可能岗位匹配度较高，但人才评级一般。


### 二、适用范围

| 项目 | 说明 |
| --- | --- |
| 适用阶段 | 简历筛选、岗位推荐、转岗匹配、人才池二次分发 |
| 适用对象 | 所有岗位、所有职级、所有候选人类型 |
| 不适用于 | 最终录用决策、薪资定级、绩效评估 |
| 评分依据 | 简历、作品集、公开资料、推荐信息、岗位 JD |
| 前置条件 | 必须有明确岗位要求或岗位画像 |


### 三、评分结构


岗位匹配度总分为 **100 分**。

一级维度有不同权重；每个二级维度按 **0-10 分**评分，再按二级权重折算到一级维度，最后汇总为岗位匹配度总分。

### 计算公式


| 一级维度得分 = 二级维度加权平均分 ÷ 10 × 一级维度权重   岗位匹配度总分 = 所有一级维度得分相加 |
| --- |


### 一级维度总表

| 一级维度 | 一级权重 |
| --- | --- |
| 硬性条件匹配 | 15% |
| 岗位核心能力匹配 | 30% |
| 经验场景匹配 | 20% |
| 业务/行业理解匹配 | 10% |
| 产出成果匹配 | 10% |
| 工作方式与组织适配 | 10% |
| 风险与成本匹配 | 5% |
| 合计 | 100% |


### 四、通用 10 分评分标准


| 分数 | 匹配程度 | 通用定义 |
| --- | --- | --- |
| 9-10 | 极高匹配 | 与岗位要求高度一致，且有强证据支持 |
| 7-8 | 高匹配 | 满足主要要求，能较快进入角色 |
| 5-6 | 中等匹配 | 满足部分要求，但存在明显补足空间 |
| 3-4 | 低匹配 | 只有弱相关经验或间接信号 |
| 1-2 | 明显不匹配 | 与岗位要求差距较大，或存在明显反向信号 |
| 0 | 不符合 | 不满足关键门槛或存在严重风险 |


### 五、证据等级规则


每个二级维度除了打 0-10 分，还要标记证据等级。


| 证据等级 | 定义 | 示例 |
| --- | --- | --- |
| 强 | 有明确岗位相关项目、技能、数据、结果、作品、客户案例或第三方证明 | 做过同类岗位、同类场景，并有量化成果 |
| 中 | 经历较相关，但结果、深度或贡献边界不够完整 | 有类似岗位经验，但行业/阶段不同 |
| 弱 | 只有关键词、间接经历或泛化表达 | 写了“熟悉 SaaS/AI/增长”，但没有案例 |
| 缺失 | 简历未体现该岗位要求 | 未提到相关技能或经验 |
| 负面 | 存在明显不匹配或风险 | 薪资明显超预算、无法到岗、关键技能缺失 |


### 缺失信息处理


| 情况 | 建议评分 |
| --- | --- |
| 强证据 | 8-10 |
| 中等证据 | 6-7 |
| 弱信号 | 4-5 |
| 未体现 | 默认 3 |
| 明显负面 | 0-2 |


注意：


| 岗位匹配度评分必须以 JD 和岗位画像为基准。   简历没有体现某项岗位要求，不能默认中等匹配。 |
| --- |

### 八、二级维度评分手册


### 1. 硬性条件匹配


### 1.1 学历/专业/资质


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 候选人的学历、专业背景、资格证书是否满足岗位要求 |
| 高分 8-10 | 完全满足或超过岗位要求，且背景与岗位高度相关 |
| 中分 5-7 | 基本满足要求，但专业相关性或资质完整度一般 |
| 低分 1-4 | 与岗位要求存在明显差距，但可通过经验部分弥补 |
| 缺失处理 | 关键学历/资质未体现给 3；明确不满足给 0-2 |
| 正向证据 | 学历、专业、证书、执照、行业资质、研究方向 |
| 负向证据 | 关键资质缺失、专业明显不相关且无补偿经验 |


### 1.2 工作年限


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 候选人的工作年限是否符合岗位级别要求 |
| 高分 8-10 | 年限与岗位级别高度匹配，经验成熟度合适 |
| 中分 5-7 | 年限略低或略高，但可接受 |
| 低分 1-4 | 年限明显不足或明显过高，存在适配风险 |
| 缺失处理 | 工作时间线不清给 3 |
| 正向证据 | 工作起止时间清晰，岗位级别与年限匹配 |
| 负向证据 | 年限虚高、时间线冲突、明显高配低用或低配高用 |


### 1.3 地域/到岗方式


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 是否满足办公城市、远程/混合办公、出差、到岗周期等要求 |
| 高分 8-10 | 完全满足地域和到岗要求，可按岗位节奏推进 |
| 中分 5-7 | 基本满足，但有搬迁、出差或到岗周期不确定 |
| 低分 1-4 | 地域或到岗方式存在明显不匹配 |
| 缺失处理 | 未体现给 3-4，待 HR 确认 |
| 正向证据 | 当前城市、可迁移、可出差、立即到岗、远程经验 |
| 负向证据 | 明确不接受城市/出差/办公方式 |


### 1.4 语言/证书/合规要求


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 是否满足岗位所需语言能力、证书、行业合规或特殊资质要求 |
| 高分 8-10 | 完全满足关键语言、证书或合规要求 |
| 中分 5-7 | 基本满足，但熟练度或证明材料有限 |
| 低分 1-4 | 关键要求缺失或无法验证 |
| 缺失处理 | 若为关键门槛，缺失给 2-3；非关键门槛给 4 |
| 正向证据 | 英语/多语能力、行业证书、安全/金融/医疗等资质 |
| 负向证据 | 岗位强要求但简历未体现或明确不满足 |


### 1.5 薪资/职级预期


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 候选人的薪资、职级、管理范围预期是否与岗位预算匹配 |
| 高分 8-10 | 薪资、职级、职责范围与岗位预算高度匹配 |
| 中分 5-7 | 有一定差距，但可通过谈判或岗位调整解决 |
| 低分 1-4 | 明显超预算、职级预期不匹配或管理范围诉求冲突 |
| 缺失处理 | 未体现给 4，待 HR 确认 |
| 正向证据 | 期望薪资、当前职级、管理范围、目标岗位偏好 |
| 负向证据 | 明显高配低投、预算严重不匹配、只接受管理岗但岗位为 IC |


### 1.6 基础稳定性


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 是否存在影响岗位推进的基础履历稳定性风险 |
| 高分 8-10 | 履历稳定，跳槽逻辑清楚，无明显基础风险 |
| 中分 5-7 | 有少量短期经历或空窗，但可解释 |
| 低分 1-4 | 频繁跳槽、长期空窗、劳动关系复杂，可能影响岗位稳定 |
| 缺失处理 | 根据时间线直接判断；信息不足给 3-4 |
| 正向证据 | 每段经历有合理周期，转岗逻辑清楚，关键阶段稳定投入 |
| 负向证据 | 连续短期经历、空窗无解释、职业方向反复 |


### 2. 岗位核心能力匹配


### 2.1 核心专业技能


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 是否具备岗位最关键的技能、工具、技术栈或方法论 |
| 高分 8-10 | 掌握岗位关键技能，并在真实复杂场景中产生结果 |
| 中分 5-7 | 掌握部分关键技能，能完成常规任务 |
| 低分 1-4 | 关键技能明显缺失或仅停留在了解阶段 |
| 缺失处理 | 关键技能未体现给 2-3 |
| 正向证据 | 技术栈、工具链、方法论、岗位关键能力、项目应用 |
| 负向证据 | 只列关键词，无使用场景；关键技能缺失 |


### 2.2 关键任务能力


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 是否做过岗位最重要的 2-3 类核心任务 |
| 高分 8-10 | 多次独立完成与岗位高度一致的核心任务 |
| 中分 5-7 | 做过类似任务，但复杂度、独立性或完整度一般 |
| 低分 1-4 | 只有间接相关任务经验 |
| 缺失处理 | 默认 3 |
| 正向证据 | 与 JD 核心职责高度一致的项目或任务 |
| 负向证据 | 只做边缘支持工作，与岗位核心任务不相关 |


### 2.3 专业深度


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 是否具备该岗位所需的专业深度和复杂问题处理能力 |
| 高分 8-10 | 能独立处理岗位中高复杂度问题，具备专家或负责人信号 |
| 中分 5-7 | 有一定深度，但主要处理模块级或常规问题 |
| 低分 1-4 | 专业积累浅，难以承担岗位复杂要求 |
| 缺失处理 | 默认 3 |
| 正向证据 | 专家、owner、核心模块、技术攻关、复杂项目、深度案例 |
| 负向证据 | 经历广但浅，无法证明可独立解决复杂问题 |


### 2.4 AI 增强工作能力


| 项目 | 内容 |
| --- | --- |
| 维度定义 | 是否能用 AI 工具提升该岗位的工作效率、质量或产出上限 |
| 高分 8-10 | 已用 AI 重构该岗位相关工作流，并有明确效率或质量提升 |
| 中分 5-7 | 有 AI 工具使用经验，但应用深度一般 |
| 低分 1-4 | 只泛泛提到 AI 工具，未体现岗位场景 |
| 缺失处理 | 默认 2-3 |
| 正向证据 | AI 编程、AI 分析、Agent、自动化流程、Prompt、RAG、AI 销售/运营/设计 |
| 负向证据 | 完全未体现 AI 使用，或仅写“熟悉 ChatGPT” |

| 负向证据 | 只写“负责”但无法判断责任层级 |

### 十一、评分纪律


| 规则 | 说明 |
| --- | --- |
| 必须基于 JD | 没有岗位画像，不应进行岗位匹配度评分 |
| 区分人才强与岗位合适 | 高人才评级不等于高岗位匹配度 |
| 不奖励关键词堆砌 | 简历出现关键词但没有项目证据，不应高分 |
| 不默认中位数 | 未体现岗位要求时，按缺失规则保守评分 |
| 不只看行业标签 | 行业相同但任务不同，不等于匹配 |
| 不只看 title | title 相同但职责复杂度不同，不等于匹配 |
| 重视 AI 增强能力 | AI 公司岗位应考察候选人是否能用 AI 提升工作方式 |
| 重视迁移性 | 过往成果是否能迁移到当前岗位，比单一经历标签更重要 |
| 风险维度反向评分 | 风险越低分越高，风险越高分越低 |
| 高分低证据需复核 | 分数高但证据弱，必须进入人工或面试验证 |


### 十二、最终筛选建议


| 匹配等级 | 建议动作 |

| 基本匹配 | 进入备选池，结合人才评级判断 |
| 低匹配 | 通常不推荐当前岗位，可考虑转岗 |
| 不匹配 | 淘汰或不进入当前岗位流程 |
