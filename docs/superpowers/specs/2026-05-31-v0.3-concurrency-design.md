# V0.3 并发处理设计文档

> 版本: V1.0 | 日期: 2026-05-31 | 状态: 设计阶段 | 关联: `docs/architecture/modules.md`, `docs/roadmap.md`

## 一、目标

将当前串行 pipeline 改造为 Producer-Consumer 并发架构，支持多份简历并行处理，提升百份级别简历的吞吐量。

### 1.1 验收标准

- 100 份简历，concurrency=5 时，总耗时 < 15 分钟
- 不触发 API rate limit 错误
- 单份简历任一阶段失败不影响其余
- 汇总报告包含失败清单

---

## 二、架构

### 2.1 Producer-Consumer 管道

```
Stage 1 (1 producer)              Stage 2 (N persistent consumers)

parse a.pdf ─┐                  ┌─ Worker-1: loop { recv → evaluate → 落库 }
parse b.pdf ─┤                  ├─ Worker-2: loop { recv → evaluate → 落库 }
parse c.pdf ─┼─ bounded channel ─├─ Worker-3: loop { recv → evaluate → 落库 }
parse d.pdf ─┤                  ├─ Worker-4: loop { recv → evaluate → 落库 }
parse e.pdf ─┘                  └─ Worker-5: loop { recv → evaluate → 落库 }

                                    N = --concurrency

                                    ↑ 所有 worker 结束
                                     │
Stage 3 (1 consumer) ←──────────────┘
  SELECT evaluations JOIN pipeline_states WHERE job_run_id
  → 报告 + 排名 + 失败清单
```

- **Stage 1** — 本地 IO，单线程逐文件解析，完成后推入有界 channel
- **Stage 2** — N 个常驻 worker，每个死循环从 channel recv → evaluate → 落库 → 更新 pipeline_states
- **Stage 3** — 所有 worker 结束后，从 DB 汇聚结果，生成报告

### 2.2 并发控制

```
concurrency 来源（优先级从高到低）:
  1. CLI: --concurrency 5
  2. application.yaml: llm.concurrency: 5
  3. application.yaml: llm.concurrency: auto → 默认 3
```

- N 个 worker 即为并发度，不需要额外 Semaphore
- channel 容量 = `concurrency * 2`，Stage1 超前生产时自动背压

### 2.3 数据库连接池

启动时自动检查：

```
needed = concurrency * 2 + 2
if needed > db.max_connections:
    warn "当前并发 {concurrency}，按公式 concurrency×2+2 计算，建议连接池至少 {needed}，当前 {max_connections}"
继续执行（不阻塞）
```

`db.max_connections` 默认 20，concurrency ≤ 9 时无需调整。

### 2.4 汇总汇聚

Stage 3 不直接查 evaluations 表，而是 JOIN pipeline_states 做 run-scope 过滤：

```sql
SELECT e.* FROM evaluations e
JOIN pipeline_states ps ON ps.resume_id = e.resume_id
WHERE ps.job_run_id = $run_id
  AND ps.phase = 'evaluate'
  AND ps.status = 'success'
ORDER BY e.talent_score DESC
```

失败清单来源：

```sql
SELECT resume_id, phase, error_msg FROM pipeline_states
WHERE job_run_id = $run_id AND status = 'failed'
```

