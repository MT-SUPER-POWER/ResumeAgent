# V0.4 Token 成本追踪设计文档

> 版本: V1.0 | 日期: 2026-06-01 | 状态: 设计阶段 | 关联: `docs/architecture/modules.md`, `docs/roadmap.md`

## 一、目标

建立完整的 Token 消耗与费用追踪体系。将 LLM 模型选择从硬编码配置改为数据库驱动，支持多模型、多中转、动态定价。

### 1.1 V0.4 范围变更

原 roadmap 中 V0.4 的关键词预筛选、TopK 深度分析、Excel 导出推迟到 V1.0（前端实现）。V0.4 聚焦一件事：

- **模型管理** — DB 注册可用模型（名称、协议、价格、密钥）
- **Router 层** — 查表路由，协议适配，统一计费
- **成本追踪** — job_run 累计 token 和费用
- **加密存储** — API key 加密落库
- **CLI 管理** — model 子命令组

## 二、数据库

### 2.1 models 表（新增）

| 列 | 类型 | 说明 |
|---|---|---|
| `name` | TEXT PRIMARY KEY | 模型标识，如 `gpt-4o`、`claude-sonnet-4-6` |
| `protocol` | TEXT NOT NULL | `openai` 或 `claude` |
| `input_price` | REAL NOT NULL | 输入价格 USD / 1M tokens |
| `output_price` | REAL NOT NULL | 输出价格 USD / 1M tokens |
| `encrypted_api_key` | TEXT NOT NULL DEFAULT '' | AES-256-GCM 加密后 base64 |
| `base_url` | TEXT | nullable，默认走官方地址 |
| `enabled` | BOOLEAN NOT NULL DEFAULT true | 是否可选用 |

### 2.2 job_runs 表新增列

| 列 | 类型 | 说明 |
|---|---|---|
| `total_tokens` | INTEGER NOT NULL DEFAULT 0 | prompt + completion 总和 |
| `total_cost` | REAL NOT NULL DEFAULT 0 | 累计费用 (USD) |

### 2.3 种子数据

migrations 初始化时插入以下默认模型（仅元数据+官方参考价，`encrypted_api_key` 为空）：

| name | protocol | input_price | output_price |
|---|---|---|---|
| gpt-4o | openai | 2.5 | 10.0 |
| gpt-4o-mini | openai | 0.15 | 0.6 |
| claude-sonnet-4-6 | claude | 3.0 | 15.0 |
| claude-haiku-4-5 | claude | 0.8 | 4.0 |
| deepseek-v4-flash | claude | 0.14 | 0.28 |
| deepseek-v4-pro | claude | 0.435 | 0.87 |

用户通过 CLI 补充实际 api_key 并调整价格。

## 四、加密工具

### 4.1 `src/utils/crypto.rs`

```rust
// AES-256-GCM 加密，输出 base64
pub fn encrypt(plaintext: &str, master_key: &[u8; 32]) -> Result<String>;

// AES-256-GCM 解密，输入 base64
pub fn decrypt(ciphertext: &str, master_key: &[u8; 32]) -> Result<String>;
```

- `master_key` 来自 `crypto.master_key` 配置，启动时解析为 `[u8; 32]`
- 加密 nonce 随机生成，随密文一起编码
- 空字符串不解密，直接返回空

## 五、Router 层

### 5.1 `src/llm/router.rs`（新增）

```
router.chat(model_name, system_prompt, user_msg, require_json)
  → db.find_model(model_name)
      → 模型不存在 → Err
      → 模型 disabled → Err
  → decrypt(encrypted_api_key, master_key)
  → match protocol {
        // NOTE: 如果模型协议不是 openai 或 claude 的，请在此补充新分支
        "openai"  → openai_handler  → POST {base_url}/chat/completions
        "claude"  → claude_handler  → POST {base_url}/messages
        other     → Err("unsupported protocol: {other}")
    }
  → 从响应提取 prompt_tokens, completion_tokens
  → cost = prompt_tokens × input_price + completion_tokens × output_price
  → 返回 LlmResponse { content, prompt_tokens, completion_tokens, cost }
```

### 5.2 handler 拆分

现有 `llm/client.rs` 拆为：

| 文件 | 职责 |
|---|---|
| `llm/router.rs` | 查 DB、解密、路由、计费 |
| `llm/openai_handler.rs` | OpenAI 协议 HTTP 适配（请求格式、响应解析） |
| `llm/claude_handler.rs` | Claude 协议 HTTP 适配 |

handler 不关心定价、不计费，只返回 `{content, prompt_tokens, completion_tokens}`。计费统一在 Router 层。

## 六、CLI

### 6.1 model 子命令组

```
resume-agent model list
    列出所有模型（name / protocol / input_price / output_price / enabled）

resume-agent model add <name>
    --protocol openai|claude
    --api-key <key>
    --input-price <num>
    --output-price <num>
    [--base-url <url>]
    新增模型（api_key 自动加密落库）

resume-agent model set-price <name>
    --input-price <num> --output-price <num>
    更新定价

resume-agent model enable <name>
resume-agent model disable <name>
resume-agent model remove <name>
```

### 6.2 run 命令增加 --model

```
resume-agent run --dir ./resumes --model gpt-4o
```

不传则用第一个 enabled 的模型。

## 七、Pipeline 集成

### 7.1 启动

从 CLI 或默认取 `model_name` → Router 验证模型存在且 enabled。

### 7.2 每次 LLM 调用后

使用 SQL 原子累加，避免并发写覆盖：

```sql
-- 原子操作，不存在 read-modify-write 竞态
UPDATE job_runs
SET total_tokens = total_tokens + $1,
    total_cost = total_cost + $2
WHERE id = $3
```

放弃 SELECT + 应用层计算 + UPDATE 的模式。

### 7.3 结束时

`total_tokens` 和 `total_cost` 已是当次运行的总消耗，无需额外查询。
